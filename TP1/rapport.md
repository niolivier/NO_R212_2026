# Rapport — TP1

## Points clés

### 1. Utilisation de Flexbox pour structurer la page
J’ai appris à utiliser Flexbox pour organiser les éléments sur une ligne ou en colonne, notamment pour le header et la section hero. Cela permet d’aligner facilement les éléments et de gérer les espacements de manière propre.

### 2. Mise en page avec Grid pour les cartes
J’ai compris comment utiliser CSS Grid pour créer une grille responsive, en particulier avec repeat(auto-fit, minmax()). Cela permet d’adapter automatiquement le nombre de colonnes selon la taille de l’écran sans ajouter beaucoup de règles supplémentaires.

### 3. Importance du responsive design
J’ai retenu qu’une page doit rester lisible sur toutes les tailles d’écran. Grâce à des conditions selon la taille de l’écran et à des unités flexibles, j’ai pu adapter la navigation, les cartes et la section “à propos” pour le mobile.

## Utilisation de l'IA

### Démarche
J’ai utilisé Gemini 3 Flash pour lui poser des questions quand je bloquais sur un concept ou quand ma photo refusait de s'afficher.

### Apports et limites
L'IA m'a surtout aidée à débugger mes erreurs de débutante (les chemins de fichiers !) et à m'expliquer la logique derrière les sélecteurs. Cependant, il faut faire attention : elle propose parfois des trucs compliqués qui ne sont pas dans le cours

### Prompts significatifs
1.	"c quoi la difference entre le header le header ul et le header a" → m’a aidé à comprendre la structure HTML entre le conteneur, la liste et les liens
2.	"pourquoi j'ai pas la photo" → m’a aidé à vérifier les chemins d’images et comprendre une erreur d’affichage
3.	"am c le logo ?" → je n’avais pas compris si c’était le logo j’ai préféré verifier.
